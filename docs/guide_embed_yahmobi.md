# 指南書 — yah.mobi 実サイトへのチャットウィジェット設置

対象: 販売サイト yah.mobi に chat.yah.mobi のチャットを埋め込む（Intercom型）
状態: **設計＋手順書（chat側の追加実装1件の承認待ち）**

---

## 1. 全体像（仕組み）

```
yah.mobi の各ページ
  └─ <script src="https://chat.yah.mobi/widget.js"> … 1タグだけ
        ├─ 右下にランチャーボタンを描画（親ページ側・56px）
        └─ クリックで iframe パネル(380×560)を表示
              └─ chat.yah.mobi/widget-chat（チャット本体・認証/AIはすべてこの中）
        親⇔iframe は postMessage（開閉・未読バッジ・origin検証）
```
- 販売サイトとは **DOM/CSS/JS/Firebase 完全分離**（iframe境界）＝販売サイトを壊さない、の設計どおり。

## 2. 準備状況（✅=済 / 🔨=要実装）

| 項目 | 状態 |
|---|---|
| ローダー `https://chat.yah.mobi/widget.js` | ✅ ホスティング済（ランチャー・パネル・未読バッジ・`window.YahChat` API・モバイル対応） |
| CSP `frame-ancestors`（yah.mobi / www.yah.mobi からの埋め込み許可） | ✅ 設定済 |
| reCAPTCHA 許可ドメイン（yah.mobi / chat.yah.mobi） | ✅ 登録確認済 |
| `?lang=` で言語指定（ローダー→iframe） | ✅ 対応済 |
| **ウィジェット側の埋め込みモード（postMessage 受け口）** | 🔨 **未実装 ← 唯一の残作業** |

### 🔨 何が足りないか（chat側・私が実装）
現状のウィジェットは**自分のランチャーを内蔵**しているため、ローダー経由で開くと「パネルの中にもう1つランチャー」が出て二重になります。埋め込みモードを追加します：
1. **埋め込み検出**（iframe内 かつ `?origin=` が許可ドメイン）時は、内蔵ランチャーを消し**パネルをiframe全面表示**（開いた状態で待機）
2. ヘッダの「×」→ 親へ `yah:close` を送信（親がパネルを閉じる）
3. 未読発生 → 親へ `yah:unread` を送信（ランチャーに赤バッジ）
4. 受信メッセージは **origin 検証**（yah.mobi / www.yah.mobi のみ）
5. ローダーの既定言語を `ja`→`en` に修正（プロダクト方針と一致）

規模：クライアントのみ・約1時間。ルール/functions 変更なし。

## 3. 設置手順（実装完了後）

### Step 1: chat側デプロイ（あなた）
```
pnpm deploy:hosting
```

### Step 2: 販売サイトに1タグ追加（販売サイト側の作業）
全ページ共通のテンプレート（`</body>` 直前推奨）に：
```html
<script src="https://chat.yah.mobi/widget.js" data-lang="en" defer></script>
```
- オプション: `data-position="bottom-left"` / `data-color="#000000"` / `data-lang="ja|en|zh|ko|th|vi"`
- 🚨 chat 側からは販売サイトのコードを触らない（ガードレール）。タグ追加は販売サイトの担当作業。
- 販売サイトが多言語ページを持つ場合、ページ言語に合わせて `data-lang` を出し分けると最良（無指定なら en）。

### Step 3: 段階導入（推奨）
1. まず**1ページだけ**（例: FAQページ）に入れて動作確認
2. 問題なければ全ページ共通テンプレートへ

## 4. 動作確認チェックリスト（設置後）
- [ ] PC Chrome: ランチャー表示 → クリックでパネル → 3分岐 → AIチャット応答
- [ ] **iPhone Safari（最重要）**: 同上。※iframe内のサードパーティストレージ制限で**匿名認証が失敗する可能性**があるブラウザ。ここが通れば全環境ほぼOK
- [ ] パネルを閉じてAI返信が来る → ランチャーに**未読バッジ**
- [ ] Sign in（Google/メール）がiframe内で動くか ※ポップアップがブラウザにブロックされる場合あり→要確認
- [ ] `refund` などでCONTACTボタン → yah.mobi/contact が**新規タブ**で開く
- [ ] 販売サイトのレイアウト/クリックを邪魔しない（パネル閉時はランチャー56pxのみ）

## 5. リスクと備え（正直に）
| リスク | 対応 |
|---|---|
| **Safari のサードパーティストレージ制限**で iframe 内の Firebase 匿名認証が失敗 | Step 3 の1ページ導入で実機確認。失敗時の代替: ランチャーを「新規タブで開く」動作に切替（ローダー1行変更で可） |
| iframe 内の Google ログインポップアップがブロックされる | 同上で確認。問題あればログインのみ新規タブ化 |
| 旧 `data-lang` 既定が ja | 実装時に en へ修正 |
